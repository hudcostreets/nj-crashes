# `njdot/data/2024/` — denormalized `Crash.csv` only

First year of the new format. The per-table bulk dumps
(`Accidents`/`Drivers`/`Occupants`/`Pedestrians`/`Vehicles`) are
**no longer published**. NJ DOT now distributes a single
[`Crash.csv`](Crash.csv) (~700 MB) generated by the AASHTO dashboard.

## Why this matters

Each row in `Crash.csv` represents a crash, but with all driver,
occupant, vehicle, and pedestrian information collapsed into wide
duplicated columns (e.g. `Driver1Age`, `Driver2Age`, `Vehicle1Make`,
`Vehicle2Make`, ...). The clean PK-based JOIN model from the per-table
era is gone:

- Independent analysis of occupants, vehicles, or pedestrians is
  awkward (need to UNPIVOT the wide columns).
- High-cardinality crashes (e.g. multi-vehicle pile-ups) are
  truncated at whatever fixed column count the export uses.
- Schema discoverability is harder — column meanings live in the
  AASHTO dashboard UI rather than a fixed-width spec.

We are working with NJ DOT (DOT-BTDS) to restore the per-table layout
for 2024+. See [`../2022/`](../2022/) and [`../2023/`](../2023/) for the
historical per-table layout.

## Format

CSV with one header row. Columns are AASHTO-dashboard-defined; expect
~150-200 columns per row covering crash, driver, vehicle, occupant,
pedestrian, and roadway facts.
