# NJSP municipality-level plot data

## Problem

On municipality pages (e.g. `/c/Hudson/Weehawken`), the 5 NJSP plots show **county-level** data because their data sources (CSV files) only have county-level aggregations. A note on the page says "NJSP fatal crash data is available at the county level" but ideally we'd show muni-level data where meaningful.

## Current state

The 5 NJSP plots and their data sources:

| Plot | CSV file | Filter column |
|------|----------|---------------|
| `FatalitiesPerYearPlot` | `year-type-county.csv` | `county` (name) |
| `YtdDeathsPlot` | `ytd.csv` | `county` (name) |
| `FatalitiesPerMonthPlot` | `monthly.csv` | `county` (name) |
| `FatalitiesByMonthBarsPlot` | `month-year.csv` | `county` (name) |
| `HomicidesComparisonPlot` | `crash-homicide.csv` | `county` (name) |

All CSVs are generated by `njsp/cli/update_www_data.py`, which groups by county name only. The underlying NJSP crash records (`njsp/crashes.db`, `njsp/data/crashes.pqt`) DO have `mc` (municipality code).

## Feasibility assessment

~10,236 NJSP fatal crashes across 564 municipalities over 17 years = **avg ~1 fatal crash per municipality per year**. Municipality-level monthly breakdowns would be extremely sparse — most months would be 0 for most municipalities.

### What makes sense at muni level

- **FatalitiesPerYearPlot**: Yes — annual counts are meaningful even if small (0-10/year for most munis)
- **YtdDeathsPlot**: Marginal — cumulative YTD with 0-10 deaths total is not very informative
- **FatalitiesPerMonthPlot**: No — monthly time series would be almost all zeros
- **FatalitiesByMonthBarsPlot**: No — same sparsity issue
- **HomicidesComparisonPlot**: No — requires UCR homicide data which is county-level only

### Recommended approach

For the annual plot (`FatalitiesPerYearPlot`), add municipality breakdowns to `year-type-county.csv` (or a new `year-type-county-muni.csv`). For the other 4 plots, continue showing county data with the existing note.

## Plan

### Phase 1: Add muni data to annual fatalities

1. **Extend `update_www_data.py`** (or the annual summaries pipeline) to generate `year-type-county-muni.csv` with columns: `year, county, mc, muni_name, driver, passenger, cyclist, pedestrian, crashes`
   - Group by `(year, cc, mc)` instead of just `(year, cc)`
   - Include municipality name from `cc2mc2mn` mapping
   - Only include municipalities that have at least 1 fatal crash

2. **Update `FatalitiesPerYearPlot`** to accept optional `mc` prop
   - When `mc` is set, query from the muni-level CSV
   - Filter: `WHERE county = '${county}' AND mc = ${mc}`

3. **File size**: Current `year-type-county.csv` is ~8KB. Muni-level version would be ~50-100KB (still tiny).

### Phase 2: Better UX for county-only plots

For plots that remain county-level on muni pages:
- Group them under a "County-level data" subheading
- Improve the note: "{countyName} County data shown — NJSP fatal crash data is not available at the municipality level for these plots"
- Consider collapsing/hiding them behind a toggle on muni pages

### Phase 3: NJSP crashes table at muni level

The NJSP crashes table (`NjspCrashesSection`) already filters by `mc` via the D1 API — this works correctly at muni level. No changes needed.

## Data generation

The key file is `njsp/cli/update_www_data.py`. The crash data is loaded from `CRASHES_PQT` which has `cc` and `mc` columns. Municipality names can be looked up from `cc2mc2mn.json` (loaded by `GeoFilterContext` on the frontend, and available from `nj_crashes/muni.py` on the backend).
