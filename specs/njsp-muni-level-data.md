# NJSP municipality-level plot data

## Problem

On municipality pages (e.g. `/c/Hudson/Weehawken`), the 5 NJSP plots show **county-level** data because their data sources (CSV files) only have county-level aggregations. A note on the page says "NJSP fatal crash data is available at the county level" but ideally we'd show muni-level data where possible.

## Current state

The 5 NJSP plots and their data sources:

| Plot | CSV file | Filter column |
|------|----------|---------------|
| `FatalitiesPerYearPlot` | `year-type-county.csv` | `county` (name) |
| `YtdDeathsPlot` | `ytd.csv` | `county` (name) |
| `FatalitiesPerMonthPlot` | `monthly.csv` | `county` (name) |
| `FatalitiesByMonthBarsPlot` | `month-year.csv` | `county` (name) |
| `HomicidesComparisonPlot` | `crash-homicide.csv` | `county` (name) |

All CSVs are generated by `njsp/cli/update_www_data.py`, which groups by county name only. The underlying NJSP crash records (`njsp/crashes.db`, `njsp/data/crashes.pqt`) DO have `cc` and `mc` (municipality code).

## Plan

### Phase 1: Add municipality breakdowns to all NJSP CSVs

Extend `update_www_data.py` to generate per-municipality rows in each CSV, following the same pattern used for per-county rows. Add `cc` and `mc` columns alongside the existing `county` column.

For each CSV:
1. **`year-type-county.csv`** → add `(year, cc, mc)` grouped rows with `driver, passenger, cyclist, pedestrian, crashes` columns
2. **`ytd.csv`** → add `(cc, mc, year, day_of_year)` rows with cumulative fatalities
3. **`monthly.csv`** → add `(cc, mc, year, month)` rows with fatalities + 12mo avg
4. **`month-year.csv`** → add `(cc, mc, year, month)` rows
5. **`crash-homicide.csv`** → skip muni level (homicide data is county-level only from UCR)

New columns in each CSV: `cc` (int), `mc` (int). Existing rows get `cc=''`, `mc=''` (statewide) or `cc=N`, `mc=''` (county-level).

### Phase 2: Update NJSP plot components

Update each plot to accept `cc`/`mc` props and filter appropriately:
- When `mc` is set, filter to `WHERE cc = ${cc} AND mc = ${mc}`
- Fall back to county level when muni data is too sparse (optional UX decision)

### Phase 3: Better UX for sparse muni data

For municipalities with very few fatal crashes, the monthly plots may show mostly zeros. Options:
- Show the plot anyway (users can see the reality of sparse data)
- Add a note about small sample sizes
- `HomicidesComparisonPlot` stays county-level on muni pages (no muni homicide data)

## Data generation

Key file: `njsp/cli/update_www_data.py`. The crash data (`CRASHES_PQT`) has `cc` and `mc` columns. Municipality names from `cc2mc2mn.json`.

File size impact: Current CSVs total ~200KB. Adding muni breakdowns ≈10-20x more rows but still under a few MB total. Could also split per-county if needed.
