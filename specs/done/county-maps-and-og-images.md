# County maps, municipality maps, and statewide map

## Status: Hudson-only map exists. Need to generalize.

### Current state

- `/map/hudson` renders a Leaflet crash map with clustered markers
- Data: `www/public/plots/njdot/hudson-5yr-lls-if.json` (indexed-JSON format)
- Boundary: `www/public/hudson.geojson` (single county)
- Generated by: `Hudson Crashes.ipynb` using `nj_crashes.geo`, `nj_crashes._json`
- Components: `map/hudson/index.tsx`, `map/clusters.tsx`, `map/cluster.tsx`
- Tile layer: Stadia Maps `alidade_smooth` (dark/light variants)
- All hardcoded to Hudson

### Available geospatial data

| File | Coverage |
|------|----------|
| `Municipal_Boundaries_of_NJ.geojson` | All 564 NJ municipalities |
| `hudson.geojson` | Hudson County boundary only |
| `cc2mc2mn.json` | County/municipality code mappings |
| `crashes.parquet` | All crashes with `olat`/`olon` (6.6M+ rows) |

## Plan

### Phase 1: Generalize map component

Extract `map/hudson/` into a reusable `CrashMap` component:

```tsx
<CrashMap
    crashes={crashData}          // indexed-JSON decoded crash points
    boundary={boundaryGeoJSON}   // GeoJSON for region outline
    center={[lat, lng]}          // initial center
    zoom={12}                    // initial zoom
/>
```

- Route: `/map/:county` (and `/map/:county/:city` later)
- `App.tsx` route: `<Route path="/map/:county" element={<CountyMap />} />`
- `CountyMap` reads `:county` param, loads corresponding data

### Phase 2: Generate per-county crash data

Extend `Hudson Crashes.ipynb` into a CLI script (`nj_crashes/maps.py` or similar):

```bash
# Generate crash location data for all counties
nj-crashes generate-map-data --all
# Or specific county
nj-crashes generate-map-data --county hudson
```

For each county:
- Filter crashes by county boundary (spatial join)
- Filter to injury+fatal (severity `i` or `f`)
- Last 5 years
- Export as indexed-JSON: `www/public/data/map/{county-slug}.json`
- Export county boundary: dissolve municipalities from `Municipal_Boundaries_of_NJ.geojson`

Output: 21 JSON files + 21 boundary GeoJSONs (or inline boundaries).
