# Global geographic navigation + victim-type filter

## Goal

Replace per-plot county dropdowns with a page-level geography selector and victim-type filter bar. All plots on the page share the same geo context (state → county → municipality) and active victim-type filters.

## Current state

- Plot1 has an inline county dropdown + victim-type icon toggles
- Homicides has NJSP/DOT source toggle
- CrashPlot has severity/county/muni stacking controls
- URL routing: `/c/{county-slug}/{muni-slug}` already exists via `GeoFilterContext`
- All plot components already accept `cc`/`mc`/`county` props
- `cc2mc2mn.json` maps codes ↔ names for all counties and municipalities

## Design

### 1. Page-level geo selector

A persistent bar (below header, above plots) with:
- **Breadcrumb**: `NJ` > `Hudson County` > `Jersey City` (clickable to navigate up)
- **Geography picker**: menu/drawer/dropdown with county/muni selection (see below)

URL updates on selection: `/#section` → `/c/hudson/#section` → `/c/hudson/jersey-city/#section`

#### County picker (state view)

A visual picker showing all 21 counties, each as:
- **Silhouette icon**: the actual geographic shape of the county (derived from GeoJSON boundaries)
- **Text label**: county name below/beside the icon

Displayed as a grid, dropdown, or slide-out drawer. Clicking a county navigates to that county's page. The shapes make counties instantly recognizable without reading text (NJ residents know their county's shape).

**Data source**: County boundary GeoJSON from NJ GIS (NJGIN). Simplify geometries for icon use (low vertex count). Could be SVG paths embedded in the app or rendered from GeoJSON at build time.

#### Municipality picker (county view)

Same concept but for municipalities within the selected county:
- **Silhouette icon**: muni boundary shape within the county
- **Text label**: municipality name

Munis are smaller and more numerous (10-70 per county), so this may need a searchable list with shape icons rather than a full grid. The shapes help distinguish municipalities with similar names.

**Data source**: Municipal boundary GeoJSON from NJGIN. Per-county files already referenced in the parquet pipeline (`ymccmcs/` directory has per-county data).

### 2. Victim-type filter bar

Icons from `icons.tsx` (Driver, Passenger, Pedestrian, Cyclist) displayed as toggleable buttons:
- **Active** = full color icon
- **Inactive** = greyed out
- **All active** = default state (no filtering)
- Clicking toggles individual types; click-all / clear-all shortcuts

This filters Plot1's stacked bars (show/hide trace types) and potentially filters other plots' data.

**Brushing**: hovering a type icon highlights that type across all visible plots (bold LI, fade others). Clicking pins it. Uses pltly's existing pin/hover infrastructure but cross-plot via React context.

### 4. Data generation changes

Two CSV files currently lack muni-level data but **can** be generated from `crashes.parquet`:

#### `year-type-county.csv` → `year-type.csv` (add state + muni rows)

Currently sourced from NJSP annual report PDFs (county-only). Replace with aggregation from `crashes.parquet` which has `cc`, `mc`, and victim type fields per crash.

In `update_www_data.py`:
```python
# State level: group by year, sum types
# County level: group by (year, cc), sum types  
# Muni level: group by (year, cc, mc), sum types
```

Output format adds `cc` and `mc` columns (empty for state, mc empty for county).

#### `projected.csv` → add state + muni rows

Currently generated by `update-projections.ipynb` at county level only. The projection algorithm (YTD ratio × full-year prior) works at any geo level.

Changes:
- Add statewide row (aggregate all counties)
- Iterate `(cc, mc)` pairs from `cc2mc2mn.json` and project per-muni
- Filter out munis with too few crashes for meaningful projection (e.g., < 5 YTD)

#### `crash-homicide.csv` — no change

UCR homicide data is only available at county level. No known public source for municipal homicides in NJ. This file stays state + county only.
